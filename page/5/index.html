穿上夏装的北方,更美

 不知不觉,来北京一个月有余了.我在武汉的小伙伴们,你们过的还好吗？

 比起秋冬两季,夏天的时候遇上北方,简直好太多了.且不说入秋之后笼罩在北京上空的沙尘暴,单是夏天那一色换上绿装的树木,就会让祖国南方或者祖国中部过来的我们,少了一些不适.

 去年来京实习,住在一条人烟稀疏的街道上,加之是冬天,因此每天7、8点下班的时候,路上基本上见不到太多人,也没有川流不息的车辆.那时候,从下车的公交站牌步行回家的行程约摸是5,6分钟,在那种相对安静的环境下,这5,6分钟似乎也是可以静下来思考的,这种思考具体到待会买什么菜,娃娃菜炒肉还是伴着鸡蛋.

 今年毕业过来,我和我的朋友们租住在一个近地铁的小区里,比起去年的环境,今年这里可以说是‘进城了’.同样的下班回家也需要步行5,6分钟,但是现在下班之路,似乎并没有像去年那样会伴着思考,姑且是怪路边的车和匆匆擦肩而过的行人吵到我的思绪了吧.

静下心来,戒骄戒躁

 人是有多面的,高兴的时候会像个孩子一样,手舞足蹈；静下来之后,也可以坐在沙发上发呆,思考一些平时遇上才会思考的东西.我也是这样多面的性格,但其实内心深处,我更喜欢那个发呆的我.

 静下心来阅读于人是有好处的.因此,我对我那“除了看书之外没有优点的室友”,给十分的好评.当然更欣赏提笔写出美文,记录自己生活的同学.因为在这个网络高度普及的时代,大多年轻人的业余时间都花在浏览网页上了,太稀缺这种‘安静的美男子’了.

因为寄托,所以跨越

 上学的时候,我喜欢去爬八分山.八分山距离武昌城区20余公里,但是路途环境比较恶劣,一路带着口罩骑过去,得花上40分钟.每当我觉得生活太艰难,或者程序写不下去了的时候,我就会带着我的单车去爬八分山.八分山有着很陡的坡度,我会觉得,连这样的高山我都能踩单车越过,生活中还有什么能倒我呢？这样的方法屡试不爽,一次次的历练,我也爱上了这座距离我们宿舍有点距离的小山.离开武汉的时候,寻思能在走之前,去山上看看,可惜没有如意.

北方的八分山在哪儿呢

 蓦然回首才发现,4年上学的时间是短暂而快乐的.对于已经步入职场的我,就像前几天在应届生见面会上说的一样,希望能在工作好的同时,保持在学校的那一份童真和快乐.

 祝福小艾,祝福和小艾一起毕业的同学们.

前言

关于怎么利用Git Page 和Hexo搭建一个博客,网上已经有比较多详细的教程了,在此不再累述,只此推荐教程： Hexo搭建教程 .

注意事项：

  • 第一点要注意的,在 github.com上创建完 yourname.github.io 之后 ,记得找到仓库右侧的 Settings->GitHub Pages选项,然后点击 Automatic page generator ,然后一路确定,生成你的Github Page 单页,如图所示：

    至此,等待数分钟,你就可以通过yourname.github.io访问到你的GitHub Pages了.

  • 第二点要注意的,上传到github的时候,_config.yml中deploy下方配置github账号的时候,type：和git、repository: 和对应的字段之间记得有空格,如下：

  • 第三点要注意的当你以上两个易错的细节注意之后,执行hexo deploy的时候,输入git用户名和密码就应该自动给你push了,而不用自己手动去git add,git push等操作.

  • 最后一点是关于独立域名的.
    在github.com/youname/yourname.github.io这个仓库添加CNAME文件和内容即可实现域名绑定.
    但是每次本地执行hexo clean hexo generate 最后hexo deploy提交到github的时候你会发现CNAME被本地没有CNAME的版本覆盖了.
    在此提供一个解决办法,hexo deploy的实现机制是将nodejs工具生成在public目录下的文件copy一份到.deploy_git这里,然后再从.deploy_git这个文件夹push到github,为解决CNAME被覆盖的问题,你要做的是：

    1. 在.deploy_git目录里新建CNAME文件（内容是你的域名如stayhungry.me）,并执行git add CNAME.

    2. 在执行hexo generate之后,hexo deploy之前,复制1步骤中的CNAME文件到public目录下,紧接着再去hexo
      3. deploy的时候就可以将CNAME上传到你的仓库了.

    当然这个问题还有一个更彻底的根治方法,就是找到nodejs生成模板文件的那一段代码,然后将CNAME文件也改成每次hexo generate都自动生成.如果你已经发现了,或者有更好的方法,欢迎您将你的研究成果发送到我的邮箱hizhenghang@gmail.com,以供我学习一下,我将不甚感激.当然我也会将您的方法同步到我的Blog,并署上你的大名.

####7月26日补充：
关于CNAME文件更好的解决方案：可以将CNAME文件放在hexo/source文件夹下，这样生成的时候，nodejs就会自动将CNAME文件同步到public目录下，而省去了手动添加的麻烦。
